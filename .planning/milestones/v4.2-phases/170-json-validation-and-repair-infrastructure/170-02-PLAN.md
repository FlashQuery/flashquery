<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tool/MCP text -> macro runtime | External or model-originated tool text becomes macro values and errors. |
| Template output -> MCP caller | Host-template generated text becomes public MCP result content and structuredContent. |
| Macro result -> task registry | Macro execution output controls persisted task status transitions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-170-02-01 | Tampering | `parseToolResultPayload()` | mitigate | Repair and parse before fallback; tests assert expected-error envelope fields survive repair. |
| T-170-02-02 | Repudiation | `transitionTaskFromResult()` | mitigate | Irreparable structured task envelopes fail rather than complete, preserving accurate task status. |
| T-170-02-03 | Information Disclosure | host-template parse errors | mitigate | Use bounded `invalid_json_payload` details; do not echo large raw template/model output by default. |
| T-170-02-04 | Denial of Service | JSON-like detection | mitigate | Conservative predicate only for starts-with object/array or fenced JSON; ordinary prose remains text-only. |
| T-170-02-SC | Tampering | shared parser dependency | mitigate | Uses Plan 01 approved `jsonrepair` dependency; no new package installs in this plan. |
</threat_model>
