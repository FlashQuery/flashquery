<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP caller -> document tools | Caller inputs drive path, identifier, frontmatter, and content operations. |
| Vault filesystem -> document tools | Markdown files and frontmatter are local persisted user data. |
| Supabase -> document tools | Document rows and plugin ownership data are persisted state. |
| Embedding provider queue -> document tools | Successful writes schedule asynchronous embedding work. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-153-01 | Tampering | `write_document` move | mitigate | Preserve write locks, targeted scans, stale-row cleanup, and frontmatter conflict checks; run write document regression tests. |
| T-153-02 | Information Disclosure | `get_document` move | mitigate | Preserve existing include options and response formatting; do not add new logs containing document body. |
| T-153-03 | Denial of Service | embedding scheduling | mitigate | Preserve existing `scheduleDocumentEmbedding` calls and do not add synchronous embedding waits. |
| T-153-04 | Elevation of Privilege | plugin readonly-folder checks | mitigate | Keep plugin readonly ownership checks intact in the moved write module. |
| T-153-SC | Tampering | dependency/module structure | mitigate | Use static guards and typecheck to ensure the entrypoint and moved modules compile without cycle-prone imports. |
</threat_model>
