# Requirements: FlashQuery Core v3.6 Bug Fixes & Host Parity

## Milestone Goal

1. Make template-tool discovery bounded by suppressing ordinary non-template documents and moving discovery to indexed template metadata.
2. Bring native-tool `help: true` behavior to parity between delegated model calls and host MCP `tools/call` calls.

## Outcomes

| Area | Outcome |
|------|---------|
| Template discovery warning flood | Ordinary non-template documents no longer emit `not_template` warnings into public discovery surfaces. |
| Template metadata index | `fqc_documents.template_meta` stores template-scoped metadata and is maintained by write, copy, scan, and backfill paths. |
| Indexed discovery | Production discovery reads active indexed document rows rather than recursively walking the vault on request paths. |
| `list_purposes` shape | Permissive mode emits exposed `template_tools` once at top level; restrictive mode keeps per-purpose template tools. |
| `call_macro` | Macro template metadata inherits the indexed registry path without changing the public macro output shape. |
| Native help parity | Delegated and host native calls share the same help-aware dispatch core. |
| Brokered boundary | Brokered tools still receive `help: true` unchanged and brokered errors remain unwrapped by native footer logic. |
