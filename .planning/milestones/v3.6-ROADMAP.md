## Overview

v3.6 closed two behavior gaps from the bug backlog:

- Template discovery is now bounded and index-backed. Ordinary non-template documents are silent skips, while genuine template diagnostics remain visible.
- Native `help: true` behavior now has host/delegated parity. Host MCP `tools/call` routes exposed native tools through the same shared help-aware core used by delegated model calls, while hidden native and brokered-tool boundaries remain intact.

## Phase 144: Fix Template Warning Flood And Host Help Convention Parity

## Success Criteria

1. Template discovery treats ordinary non-template documents as silent skips, preserves genuine template diagnostics, keeps provider tool/dispatch behavior unchanged, and returns bounded `list_purposes`, `search`, `resolver:"purpose"`, and `call_macro` behavior.
2. `fqc_documents` carries template-scoped `template_meta`, populated by write and scan/backfill paths, and registry discovery uses indexed active document rows instead of recursive vault walks.
3. `list_purposes` emits exposed templates once at top level in permissive mode while retaining per-purpose template tools in restrictive mode and keeping warnings/conflicts/dangling paths scoped per purpose.
4. Native-tool dispatch uses one shared help-aware core for delegated and host paths; host `tools/call` supports `help: true`, advertises optional `help` on native schemas, gates hidden native tools by host exposure, and preserves brokered-tool pass-through semantics.
5. Tests mapped in both supplied test plans pass: Template Warning Flood T-U-001..007, T-I-001..009, T-S-001..006; Help Convention T-U-001..008 and T-E-001..010.

## Key Accomplishments

- Implemented silent-skip semantics for ordinary non-template documents, removing `not_template` noise from discovery/search/metadata surfaces while preserving genuine diagnostics.
- Added and verified `fqc_documents.template_meta JSONB`, populated and cleared through write/copy/scanner/backfill paths.
- Switched production template discovery to active indexed document rows, including macro template metadata.
- Updated `list_purposes` so permissive mode emits exposed templates once at top level and restrictive mode keeps per-purpose template lists.
- Extracted shared native dispatch into `src/llm/native-tool-core.ts`.
- Added host MCP `tools/call` native help parity with optional `help` schema advertisement, hidden native exposure gating, and brokered pass-through preservation.
