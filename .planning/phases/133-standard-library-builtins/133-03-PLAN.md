<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| macro args -> trace/progress | Macro author controls user-visible log/progress message values. |
| invocation context -> task introspection | Builtins expose task IDs and task lists to macro code. |
| async builtin duration -> evaluator | `sleep` and `slow_op` can delay execution and must observe cancellation hooks. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-133-07 | Information Disclosure | `list_tasks` | mitigate | Use invocation-owned `context.listTasks` or a current-invocation fallback; never import singleton task registry or cross-session state. |
| T-133-08 | Information Disclosure | `echo` / `status` | mitigate | Keep log and progress channels separate and assert `status` does not write log while `echo` does not call progress sink. |
| T-133-09 | Denial of Service | `sleep` / `slow_op` | mitigate | Validate non-negative finite duration and call `context.checkCancelled` every <=100ms chunk. |
</threat_model>
