<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Alias key -> resolver | Untrusted alias names control which entry is rendered. |
| `_items` array -> document resolver | Untrusted list items select multiple documents/templates. |
| MCP schema -> resolver | Public `template_params` crosses from external callers into internal hydration. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-114-09 | S/I | Alias lookup | mitigate | `@alias` must only read `templateParams[alias]`; it must never resolve `alias` through vault path or filename search. |
| T-114-10 | T/I | `_items` list | mitigate | Resolve list items in order through existing document/template helpers and wrap failures as `multi_ref_item_failed` with alias and index. |
| T-114-11 | T | `call_model` discovery resolvers | mitigate | Discovery branches return before template/reference parsing; untrusted `template_params` is ignored there. |
| T-114-12 | T/E | Reserved controls | mitigate | `_template`, `_items`, and `_separator` are resolver controls, not user params; exclude them from validation and metadata. |
</threat_model>
