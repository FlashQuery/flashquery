<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `templateParams` -> resolver | Untrusted object values control template rendering. |
| `document` param -> vault/Supabase | Untrusted identifiers select documents. |
| Rendered template -> provider prompt | Rendered markdown becomes model-visible context. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-114-05 | T/I | `document` params | mitigate | Reuse `resolveAndBuildDocument`; map not-found, ambiguity, read, and traversal failures to `template_param_doc_not_found`; never read a raw path from params. |
| T-114-06 | T | `renderTemplateContent` | mitigate | Scan only the original template body and apply replacements once right-to-left; never recursively hydrate substituted strings. |
| T-114-07 | I | Plain document references | mitigate | Branch only on `fq_template === true`; plain docs ignore `templateParams` so caller cannot alter non-template document content. |
| T-114-08 | T/E | Reserved fields | mitigate | Strip `_template`, `_items`, and `_separator` before validation and metadata so control fields cannot collide with declared params. |
</threat_model>
