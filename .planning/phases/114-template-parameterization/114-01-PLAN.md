<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP host -> call_model schema | Untrusted `template_params`, aliases, and reference text enter through MCP params. |
| Template params -> vault document resolver | Untrusted path/fq_id/filename identifiers choose vault documents. |
| Template params -> prompt content | User-provided strings become model-visible prompt context. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-114-01 | T/I | `resolveReferences` document identifiers | mitigate | Tests must require use of existing document resolver for template and document params; no raw path reads from `template_params`. |
| T-114-02 | T | Template substitution | mitigate | Tests must prove single-pass substitution and no recursive `{{ref:...}}` hydration from injected values. |
| T-114-03 | I | Document params | mitigate | Tests must force ambiguity/not-found through typed `template_param_doc_not_found` rather than selecting an arbitrary document. |
| T-114-04 | T/E | Reserved fields | mitigate | Tests must prove `_template`, `_items`, and `_separator` are consumed as resolver controls and excluded from `template_params_used`. |
</threat_model>
