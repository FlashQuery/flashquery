## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML config to runtime lock helper | Operator-controlled `locking.lock_timeout_seconds` affects liveness and database polling. |
| Lock helper to PostgreSQL | Retry loop repeatedly queries advisory lock state on a checked-out client. |
| Same-process caller to burst queue | Concurrent requests can wait behind an active burst before reaching callback execution. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-159-04 | Denial of Service | `locking.lock_timeout_seconds` | mitigate | Zod validates positive integers, default is 10, and tests reject invalid values. |
| T-159-05 | Denial of Service | Tier 1 and active burst wait | mitigate | Apply the same deadline to queued/burst/Tier 1 waits so same-process contention cannot hang. |
| T-159-06 | Repudiation | Timeout error handling | mitigate | `LockTimeoutError` carries reason `lock_timeout` for structured downstream envelopes. |
| T-159-07 | Tampering | Partial advisory acquisition | mitigate | Track acquired advisory keys only after `pg_try_advisory_lock` returns true and release in reverse in `finally`. |
| T-159-SC | Tampering | npm/pip/cargo installs | accept | No package install is planned; existing dependencies only. |
