<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP tool to embedding helper | Untrusted tool payload text becomes embedding input and pending retry state. |
| Helper to Postgres/Supabase | Failed provider/update state crosses into durable `fqc_pending_embeds`. |
| Schema DDL to hosted/local database | Startup DDL mutates user-owned Supabase schema. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-146-01 | Tampering | `fqc_pending_embeds` target fields | mitigate | Use typed target descriptors and never accept raw table names from MCP input in Plan 1. |
| T-146-02 | Information Disclosure | `embed_text` pending rows | mitigate | Store only required retry text and avoid logging raw text in `background_embed_failed`. |
| T-146-03 | Repudiation | provider failures | mitigate | Stable structured log event `background_embed_failed` plus durable attempt metadata. |
| T-146-04 | Denial of Service | pending retry table growth | mitigate | Add indexed status/next_retry selectors and attempt metadata for bounded worker behavior. |
| T-146-05 | Elevation of Privilege | cross-instance pending updates | mitigate | Include `instance_id` in unique key and all helper queries. |
| T-146-SC | Tampering | npm installs | accept | No package install in this plan; existing `pg` and Supabase dependencies only. |
| T-146-REC-SQL | Tampering | dynamic record table SQL | transfer | Dynamic table SQL is planned in 146-04 with escaped identifiers and pooled SQL boundaries. |
</threat_model>
