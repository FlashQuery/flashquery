<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Pending table to worker | Durable retry rows drive provider calls and target updates. |
| Worker to dynamic record tables | Retry updates may touch plugin-owned record tables. |
| Doctor to operator output | Diagnostic data leaves database context for terminal output. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-146-11 | Spoofing | worker target selection | mitigate | Scope pending row selection and target updates by `instance_id`. |
| T-146-12 | Tampering | dynamic record retry SQL | mitigate | Reuse target abstraction and escaped table identifiers. |
| T-146-13 | Information Disclosure | doctor output | mitigate | Report counts/ids/labels only; never print raw `embed_text` or document bodies. |
| T-146-14 | Denial of Service | retry loop | mitigate | Use bounded `limit`, due-time filtering, and attempt metadata. |
| T-146-15 | Repudiation | repeated provider failures | mitigate | Retain `attempt_count`, `last_error`, and `last_attempt_at` on failed retries. |
| T-146-SC | Tampering | npm installs | accept | No package install in this plan. |
</threat_model>
