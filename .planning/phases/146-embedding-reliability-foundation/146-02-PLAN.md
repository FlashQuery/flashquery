<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP clients to tool handlers | Untrusted write content becomes embedding input and public response JSON. |
| Tool handlers to helper | Tool code passes target metadata into durable helper state. |
| Scenario runner to managed server | Test-only failure injection must not become production behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-146-06 | Information Disclosure | pending rows and logs | mitigate | Do not log raw document, memory, or record content in helper failure logs. |
| T-146-07 | Spoofing | cross-instance target updates | mitigate | Tool targets include `config.instance.id`; helper updates filter by `instance_id` where target supports it. |
| T-146-08 | Repudiation | silent provider failures | mitigate | Public warnings plus durable pending rows replace log-only failure paths. |
| T-146-09 | Tampering | dynamic record table metadata | mitigate | Use existing plugin table resolution; do not accept raw table names from MCP input. |
| T-146-10 | Denial of Service | public write latency | mitigate | Preserve foreground write success semantics and avoid making provider success mandatory. |
| T-146-SC | Tampering | npm installs | accept | No package install in this plan. |
</threat_model>
