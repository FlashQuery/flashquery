<threat_model>
Threats:
- Metadata drift could expose unsafe tools to delegated models.
- Incorrect hard-exclusion handling could allow recursive `call_model` or administrative plugin tools into delegated tool belts.
- Legacy-name suggestions could silently rewrite removed tools instead of failing clearly in later phases.

Mitigations:
- Unit tests must assert delegated hard exclusions win over explicit delegated tool names.
- Metadata tests must assert no registered current tool is missing metadata.
- Legacy suggestion helpers must return suggestions only; no automatic aliasing is allowed.
</threat_model>

  <action>
Create `src/mcp/tool-metadata.ts` as the canonical source for MCP tool metadata.

Define these exported types:
```typescript
export type ToolCategory = 'doc-read' | 'doc-write' | 'memory' | 'plugin' | 'llm' | 'system';
export type ToolTier = 'read-only' | 'read-write' | 'admin';
export type ToolStatus = 'final' | 'transitional' | 'removed' | 'dead';
```

Define `ToolMetadata` with fields:
```typescript
name: string;
status: ToolStatus;
categories: ToolCategory[];
tier: ToolTier;
hostEligible: boolean;
delegatedEligible: boolean;
delegatedHardExcludedReason?: string;
legacyNames?: string[];
replacement?: string;
description: string;
```

Define `TOOL_METADATA` entries for all current registered tools plus final/removed names from the v3.3 requirements. At minimum include current tools from `src/mcp/tools/*.ts`, final target tools named in `.planning/REQUIREMENTS.md`, removed/dead names (`append_to_doc`, `create_document`, `update_document`, `update_doc_header`, `search_documents`, `save_memory`, `update_memory`, `search_memory`, `list_memories`, `list_projects`, `get_project_info`, etc.), and transitional tools (`get_briefing`, `insert_doc_link`).

Export pure helpers:
```typescript
getToolMetadata(name: string): ToolMetadata | undefined
requireToolMetadata(name: string): ToolMetadata
listToolMetadata(filter?: Partial<Pick<ToolMetadata, 'status' | 'hostEligible' | 'delegatedEligible'>>): ToolMetadata[]
getToolNamesByTier(tier: 'tier:read-only' | 'tier:read-write'): string[]
getDelegatedHardExcludedTools(): Array<{ tool: string; reason: string }>
getLegacyToolSuggestion(name: string): { replacement: string; message: string } | undefined
```

Descriptions must follow the XC-8 four-block template with literal labeled blocks:
`Summary:`, `Use when:`, `Do not use when:`, and `Example:`.
  </action>
  <acceptance_criteria>

    - `src/mcp/tool-metadata.ts` contains `delegatedHardExcludedReason`.
    - `npm test -- tests/unit/tool-metadata.test.ts` exits 0 after tests are added.
  </acceptance_criteria>
