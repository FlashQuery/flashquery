<threat_model>
Threats:
- JSON helpers could accidentally emit prose strings that downstream agents cannot parse.
- Expected errors could keep using `isError: true`, causing callers to treat user/input failures as runtime failures.
- Identification builders could omit fields needed for follow-up calls.

Mitigations:
- Unit tests parse every helper result with `JSON.parse(result.content[0].text)`.
- Expected-error tests assert `isError` is exactly `false` or omitted according to helper contract.
- Builder tests assert required fields from the product doc for every entity type.
</threat_model>

Required exports:
```typescript
export type CanonicalErrorCode = 'not_found' | 'ambiguous_identifier' | 'permission_denied' | 'invalid_input' | 'conflict' | 'unsupported' | 'not_supported_in_mode';
export type WarningCode = string;
export interface ErrorEnvelope { error: CanonicalErrorCode | string; message: string; identifier?: string; details?: Record<string, unknown>; }
export interface ToolResult { content: Array<{ type: 'text'; text: string }>; isError?: boolean; }

export function jsonToolResult(payload: unknown): ToolResult;
export function jsonExpectedError(error: ErrorEnvelope): ToolResult;
export function jsonRuntimeError(message: string, details?: Record<string, unknown>): ToolResult;
export function withWarnings<T extends Record<string, unknown>>(payload: T, warnings: WarningCode[]): T & { warnings?: WarningCode[] };
export function batchResult(results: unknown[]): unknown[];
```

Implement identification builders with required fields:
- Document: `{ identifier, title, path, fq_id, modified, size: { chars } }`
- Memory: `{ memory_id, content_preview, tags, plugin_scope, created_at, updated_at }`
- Record: `{ id, plugin_id, table, created_at, updated_at }`
- Plugin: `{ plugin_id, name, status, table_count }`
- LLM call: `{ resolver, name, resolved_model_name, provider_name }`
