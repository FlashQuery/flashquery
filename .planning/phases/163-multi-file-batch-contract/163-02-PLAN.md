## Trust Boundaries

| Boundary | Description |

| MCP caller -> archive/remove handlers | Untrusted identifiers and tokens enter destructive file operations. |
| Handler -> vault filesystem | Batch items can archive, trash, or delete real vault files. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-163-02 | Tampering | repeated/path-like identifiers | mitigate | Use object-form token co-location and normalized ordered items; do not introduce maps or positional arrays. |
| T-163-03 | Tampering | archive/remove stale writes | mitigate | Compare item.version_token against fresh locked disk bytes before mutation; return conflicted on mismatch. |
| T-163-04 | Repudiation | partial batch result reporting | mitigate | Wrap every batch item as succeeded, conflicted, or failed in input order. |
| T-163-05 | Tampering | best-effort batch semantics | mitigate | Keep item processing independent and assert successful writes persist after failures; do not add rollback. |
| T-163-SC | Tampering | npm installs | accept | No package installs are planned for this phase. |
</threat_model>
