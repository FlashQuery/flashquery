<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `.tool.md` filesystem to startup registry | Local source metadata becomes model-visible descriptions and help content. |
| model args to native dispatcher | Untrusted model args may request `help:true` before normal schema validation. |
| brokered upstream errors to model | External server errors cross through FlashQuery without native decoration. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-141-03 | Tampering | `.tool.md` loader | mitigate | Static source-tree glob, Zod validation, duplicate/name mismatch fail-fast. |
| T-141-04 | Elevation of Privilege | help sentinel | mitigate | Sentinel branch applies only after native tool visibility/catalog lookup and only for FQ-native dispatch. |
| T-141-05 | Spoofing | brokered errors | mitigate | Brokered errors and brokered `help:true` are not wrapped with FQ-native help text. |
| T-141-SC | Tampering | npm installs | accept | Uses existing `gray-matter` dependency; no new package install. |
</threat_model>
