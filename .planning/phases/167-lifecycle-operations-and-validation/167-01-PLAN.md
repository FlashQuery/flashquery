<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP caller to maintain_vault | Untrusted tool input controls lifecycle action names, scope, confirm, max_rows, and job ids. |
| Service to PostgreSQL | Lifecycle jobs and locks persist in database state and control destructive or long-running work. |
| Status/abort lookup | Caller-provided job_id crosses into persisted job lookup. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-167-01-01 | Tampering | `src/mcp/tools/scan.ts` lifecycle schema | mitigate | Validate all new parameters with Zod and action-specific semantic validation before service dispatch. |
| T-167-01-02 | Denial of Service | `max_rows` lifecycle processing | mitigate | Enforce pre-work `max_rows` ceiling in `validateMaxRows`; negative values refused; rebuild requires an explicit cap. |
| T-167-01-03 | Tampering | `fqc_maintenance_jobs` lock rows | mitigate | Use partial unique index plus heartbeat recovery; lifecycle processors release or mark jobs terminal in success, failure, and abort paths. |
| T-167-01-04 | Repudiation | background lifecycle jobs | mitigate | Persist job id, action, embedding name, timestamps, status, counts, failures, and error envelope for status polling. |
| T-167-01-SC | Tampering | package installs | accept | No npm/pip/cargo installs are planned; if an executor adds one, stop and run the package legitimacy gate from RESEARCH.md first. |
