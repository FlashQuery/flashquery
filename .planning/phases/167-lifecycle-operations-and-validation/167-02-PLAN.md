## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP caller to status/abort | Caller-provided `job_id` controls durable job lookup and abort signaling. |
| Service to PostgreSQL | Lifecycle jobs and locks persist in database state and control destructive or long-running work. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-167-02-01 | Tampering | `fqc_maintenance_jobs` lock rows | mitigate | Use partial unique index plus heartbeat recovery; lifecycle processors release or mark jobs terminal in success, failure, and abort paths. |
| T-167-02-02 | Repudiation | background lifecycle jobs | mitigate | Persist job id, action, embedding name, timestamps, status, counts, failures, and error envelope for status polling. |
| T-167-02-03 | Tampering | abort dispatch | mitigate | Reject embedding-specific parameters and update only running durable lifecycle jobs by `job_id`. |
| T-167-02-SC | Tampering | package installs | accept | No npm/pip/cargo installs are planned; if an executor adds one, stop and run the package legitimacy gate from RESEARCH.md first. |
</threat_model>
