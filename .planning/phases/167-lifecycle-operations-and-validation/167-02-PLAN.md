<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP caller to lifecycle processor | Caller controls scope, embedding_name, dry_run, background, confirm, stale_only, and mismatched_width_only. |
| Lifecycle processor to embedding providers | Row content crosses to configured embedding endpoints. |
| Lifecycle processor to PostgreSQL | Processor writes vectors/stamping columns and reindexes pgvector indexes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-167-02-01 | Tampering | `runBackfillEmbeddings` row selection | mitigate | Select only `embedding_<name> IS NULL`; never overwrite populated rows in backfill. |
| T-167-02-02 | Information Disclosure | provider calls | mitigate | Reuse existing provider path; do not log row text, embedding vectors, API keys, or provider response bodies beyond sanitized errors. |
| T-167-02-03 | Denial of Service | rebuild | mitigate | Require `confirm` and `max_rows`; enforce count cap before provider calls. |
| T-167-02-04 | Tampering | vector/stamp writes | mitigate | Use `updateTargetEmbedding` with `EmbeddingWriteStamp` to write vector and stamp columns atomically. |
| T-167-02-SC | Tampering | package installs | accept | No npm/pip/cargo installs are planned; if an executor adds one, stop and run the package legitimacy gate from RESEARCH.md first. |
</threat_model>
