<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP caller to records lifecycle | Caller controls lifecycle scope and can attempt to override plugin embedding choices. |
| Records lifecycle to plugin registry | Plugin-authored/registered metadata controls record table embedding choice. |
| Lifecycle processor to embedding providers | Plugin record content crosses to configured embedding endpoints. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-167-05-01 | Elevation of Privilege | records scope | mitigate | Use frozen plugin registration choices; reject pure-records top-level embedding_name to prevent caller override of plugin policy. |
| T-167-05-02 | Tampering | record vector writes | mitigate | Use `recordEmbeddingTarget` and `updateTargetEmbedding` with `EmbeddingWriteStamp`; do not write ad hoc SQL for vectors. |
| T-167-05-03 | Information Disclosure | provider calls | mitigate | Reuse existing provider path; do not log row text, embedding vectors, API keys, or provider response bodies beyond sanitized errors. |
| T-167-05-SC | Tampering | package installs | accept | No npm/pip/cargo installs are planned; if an executor adds one, stop and run the package legitimacy gate from RESEARCH.md first. |
</threat_model>
