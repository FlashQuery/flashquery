<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP caller to retire | Caller controls destructive retire action and confirm value. |
| Lifecycle retire to PostgreSQL DDL | Retire executes DROP FUNCTION, DROP INDEX, DROP COLUMN, and DELETE operations. |
| Records lifecycle to plugin registry | Plugin-authored/registered metadata controls record table embedding choice. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-167-03-01 | Tampering | `runRetireEmbedding` | mitigate | Require `confirm === embedding_name`; reject dry_run/background/scope/max_rows before DDL; validate identifiers with existing safe SQL helpers. |
| T-167-03-02 | Tampering | retire transaction | mitigate | Execute all per-entry artifact drops and catalog deletion in one PostgreSQL transaction. |
| T-167-03-03 | Denial of Service | plugin retire conflict | mitigate | Refuse retire while any plugin registry row stores the target embedding_name and list affected plugin IDs. |
| T-167-03-04 | Elevation of Privilege | records scope | mitigate | Use frozen plugin registration choices; reject pure-records top-level embedding_name to prevent caller override of plugin policy. |
| T-167-03-SC | Tampering | package installs | accept | No npm/pip/cargo installs are planned; if an executor adds one, stop and run the package legitimacy gate from RESEARCH.md first. |
</threat_model>
