## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| delegated model -> tool dispatcher | Model-produced tool names and arguments are untrusted input. |
| delegated model -> reference syntax | Model-produced `{{ref:...}}` strings must remain data. |
| loop executor -> cost tracker | Multi-round usage must collapse to one aggregate accounting record. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-117-01 | Elevation of Privilege | dispatcher registry lookup | mitigate | Tests require dispatch only from immutable per-call `nativeToolNames`, preserving Phase 116 hard exclusions for `call_model` and admin/plugin tools. |
| T-117-02 | Tampering | tool argument validation | mitigate | Tests require registered schema validation before handler invocation and recoverable JSON tool errors on validation failure. |
| T-117-03 | Information Disclosure | model-produced `{{ref:...}}` | mitigate | Tests require no reference hydration for assistant messages, tool results, or tool-call arguments. |
| T-117-04 | Denial of Service | retry/cost spirals | mitigate | Tests require `timeout_ms`, `max_iterations`, `max_tokens_budget`, and `max_cost_usd` stops before the next model call. |
| T-117-05 | Repudiation/Tampering | usage aggregation | mitigate | Tests require one aggregate row and calls-log token sums equal public metadata token totals. |
</threat_model>
