<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| executor -> provider | Loop sends accumulated messages and tools to provider each round. |
| model output -> executor | Assistant tool calls and text determine next loop action. |
| executor -> cost tracker | Aggregate usage crosses into persistent usage storage. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-117-01 | Elevation of Privilege | `executeAgentLoop` dispatch path | mitigate | Executor uses dispatcher from Plan 02 and immutable registry snapshot; no recursive `call_model` path is introduced. |
| T-117-02 | Tampering | tool argument path | mitigate | Executor never invokes handlers directly; dispatcher validates arguments. |
| T-117-03 | Information Disclosure | message hydration boundary | mitigate | Executor never calls reference resolver; model-produced `{{ref:...}}` remains literal data. |
| T-117-04 | Denial of Service | loop guardrails | mitigate | Check timeout/iterations/tokens/cost before next model calls, propagate `AbortSignal` into dispatch, and stop cleanly for dispatch-time timeout and cooperative shutdown. |
| T-117-05 | Repudiation/Tampering | usage accounting | mitigate | Use non-recording chat per iteration and exactly one aggregate `recordLlmUsage()` call. |
</threat_model>
