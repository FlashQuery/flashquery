<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| delegated model -> dispatcher | Tool names and arguments are untrusted. |
| dispatcher -> native handlers | Validated input crosses into FlashQuery storage/memory/document tools. |
| tool result -> loop messages | Handler output is serialized back to a delegated model. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-117-01 | Elevation of Privilege | `dispatchNativeToolCall` | mitigate | Dispatch only when `toolCall.function.name` is present in immutable `nativeToolNames` and catalog; never call `call_model` unless Phase 116 exposed it, which it never does. |
| T-117-02 | Tampering | argument validator | mitigate | Use captured Zod schema/raw shape to parse model arguments before handler invocation. |
| T-117-03 | Information Disclosure | result serialization | mitigate | Do not resolve `{{ref:...}}` in model arguments or tool results; only stringify payloads. |
| T-117-04 | Denial of Service | parallel batch | mitigate | Pass `NativeToolDispatchContext.signal` into every handler and check it before/after dispatch so executor timeout and cooperative shutdown interrupt tool execution; use all-settled batch dispatch so one abort/error does not hide sibling outcomes. |
| T-117-05 | Repudiation | dispatch log entries | mitigate | Emit structured per-call log data for later `calls_log` accounting and diagnostics. |
</threat_model>
