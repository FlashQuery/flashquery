| Boundary | Description |
|----------|-------------|
| MCP get_document caller -> vault read/repair flow | Untrusted identifier can resolve to a vault file and trigger metadata repair side effects. |
| Repair write flow -> filesystem | Repaired frontmatter bytes cross into durable disk state through writeVaultFile. |
| Repair write flow -> Supabase fqc_documents | Post-repair content_hash update must match disk bytes and returned token. |

| T-164-02 | Denial of Service | get_document cache-hit read path | mitigate | Preserve T-U-037 and source guards so pure reads do not block on document or directory locks. |

| T-164-SC | Tampering | npm installs | accept | No package installs in this plan; package legitimacy audit says no external packages. |
