<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Delegated model to dispatcher | Model-supplied tool names and arguments cross into FlashQuery internal dispatch. |
| Template dispatcher to vault/document resolver | Model arguments may resolve vault documents through `document` params. |
| Dispatcher to model result message | Hydrated template content is returned to the delegated model. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-118-12 | Spoofing | `dispatchToolCalls` | mitigate | Resolve template calls only through the per-call `templateReverseMap`; absent generated names return `tool_not_in_registry`. |
| T-118-13 | Tampering | tool arguments | mitigate | Validate arguments against `fq_params` and existing Zod/schema rules before hydration. |
| T-118-14 | Information Disclosure | template result messages | mitigate | Only templates in the current purpose registry can be invoked; document params use standard authorization/path checks. |
| T-118-15 | Denial of Service | mixed batch dispatch | mitigate | Preserve `Promise.allSettled`, abort signal checks, summary truncation, and loop guardrails. |
</threat_model>
