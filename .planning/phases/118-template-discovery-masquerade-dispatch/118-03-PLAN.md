## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Registry assembly to provider request | User-controlled tool metadata becomes provider-visible tool definitions. |
| Discovery resolver to host | Diagnostics expose template availability and path identities to callers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-118-08 | Spoofing | Combined registry | mitigate | Reserve generated `flashquery.*` names for template tools and reject final-name collisions before provider invocation. |
| T-118-09 | Repudiation | Collision diagnostics | mitigate | Include every source for a colliding generated name, including canonical `template_path` values. |
| T-118-10 | Information Disclosure | `list_purposes` diagnostics | accept | Diagnostics reveal canonical template paths already bound or globally exposed; do not include template body content. |
| T-118-11 | Denial of Service | discovery resolver | mitigate | Use existing bounded registry assembly and do not invoke providers during discovery. |
</threat_model>
