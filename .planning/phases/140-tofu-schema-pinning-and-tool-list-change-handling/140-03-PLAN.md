<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| broker TOFU state -> macro engine | Broker safety state terminates macro execution for user mediation. |
| user decision payload -> broker resolver | User-controlled approval/rejection data changes callable broker state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-140-07 | Elevation of Privilege | approval resolver | mitigate | Accept only `approve` or `reject`, scoped to pending `<server>__<tool>` decisions. |
| T-140-08 | Repudiation | TOFU approval/rejection | mitigate | Emit structured audit event with old/new hashes and trace ID for every decision. |
| T-140-09 | Denial of Service | repeated drift prompts | accept | REQ-048 explicitly forbids v1 rate limiting; blocked state still prevents execution. |
| T-140-10 | Spoofing | macro needs_user_input payload | mitigate | Payload is broker-generated and includes server/tool plus old/new schema snapshots. |
| T-140-SC | Tampering | npm installs | accept | No package installs. |
</threat_model>
