<interfaces>
Existing contracts to preserve:
- `hashToolSchema(tool: { name: string; description?: string; inputSchema?: unknown }): string` hashes upstream name, description, and inputSchema only.
- `ToolRegistry.registerTool(input)` applies `descriptionOverride` to `description` while retaining `upstreamDescription`.
- `ToolRegistry.listToolsForConsumer(ctx)` is the callable gate used by macro and agent-loop dispatch.

New contracts to create:
- `diffToolSnapshots(oldTools, newTools)` returns `{ added, changed, removed, unchanged }` where identity is `(serverId, toolName)` and change is a TOFU hash/schema hash mismatch.
- `TofuStore` or equivalent in-memory store tracks trusted schema/hash, pending schema/hash, blocked status, and retained removal tombstones keyed by `<serverId>:<toolName>`.
- `ToolIndexSink` or equivalent synchronous interface has `addTools(tools)` and `removeTools(keys)`; Phase 140 default is no-op, Phase 141 will attach BM25.
</interfaces>
