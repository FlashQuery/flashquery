<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| resolved Markdown text -> fence extractor | Untrusted document contents are scanned for executable macro blocks. |
| `source_ref` string -> selector parser | Untrusted caller selector text chooses a macro block name. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-131-04 | Tampering | `extractMacroFences` | mitigate | Accept only bare `fqm` and `fqm name=<identifier>`; malformed attributes become `parse_error/malformed_fence_attributes` before macro parsing. |
| T-131-05 | Spoofing | `selectMacroSourceBlock` | mitigate | Ambiguous, missing, duplicate, and zero-block selectors return structured `invalid_input` reasons instead of silently choosing a block. |
| T-131-06 | Information Disclosure | `source-ref.ts` | accept | Availability data lists only block names/unnamed count already present in the caller-resolved document text; document permission checks are Phase 138. |
| T-131-07 | Denial of Service | line scanner | mitigate | Line-based single pass over Markdown; no recursive Markdown parsing or regex over full document for nested structures. |
</threat_model>
