<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML config -> process startup | User-controlled YAML selectors determine the host MCP surface. |
| Startup diagnostics -> stdio MCP transport | Warning output must not corrupt stdout JSON-RPC protocol. |
| Selector strings -> metadata registry | Unknown or legacy names must not silently expand to an empty set. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-122-01 | Tampering | `src/config/loader.ts` | mitigate | Strict Zod schema and `resolveHostToolExposure` validation reject malformed `host_mcp_tools` before server startup. |
| T-122-02 | Elevation of Privilege | `src/mcp/tool-exposure.ts` | mitigate | Treat host filtering as token/context control only; do not claim authorization. Unknown and host-ineligible selectors fail validation. |
| T-122-03 | Information Disclosure | warning output | mitigate | Store warnings on config and rely on stderr logger path; no `console.log` or `process.stdout` writes from loader. |
| T-122-04 | Repudiation | traceability | mitigate | `TRACEABILITY.md` maps each CFG requirement to concrete evidence targets before coding. |
</threat_model>
