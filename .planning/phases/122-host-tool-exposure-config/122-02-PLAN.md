<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Config -> MCP SDK registration | Host exposure config controls which tool handlers are registered. |
| MCP server -> host LLM | `listTools` is the host-visible surface and must not include filtered tools. |
| Host filter -> delegated catalog | Native catalog must not contain filtered tools that delegated assembly could later reuse. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-122-04 | Information Disclosure | `listTools` | mitigate | Skip filtered names before `originalRegisterTool`; E2E asserts absent tools are not listed. |
| T-122-05 | Tampering | `getNativeToolCatalog` | mitigate | Registration gate skips catalog push and SDK call together, preventing registered/catalog drift. |
| T-122-09 | Denial of Service | skipped `registerTool` return | mitigate | Unit tests prove grouped registration functions do not throw when filtered registrations return inert values. |
| T-122-10 | Spoofing | host filtering semantics | accept | Filtering makes tools appear absent to the host LLM for token savings, but this is not authorization; do not document as security. |
</threat_model>
