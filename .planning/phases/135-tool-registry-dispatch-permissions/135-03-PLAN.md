<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| parsed macro AST -> evaluator execution | User-authored tool references become potential privileged tool invocations. |
| template masquerade metadata -> macro registry | A vault template masquerading as a tool could otherwise become callable from a macro. |
| delegated macro -> `fq.call_model` | Delegated output could recurse into another model call if not blocked. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-135-10 | Tampering | `preScanToolReferences` | mitigate | Full AST pre-scan before `execBlock` rejects forbidden/unknown references before side effects. |
| T-135-11 | Elevation of Privilege | `evalToolCall` | mitigate | Preserve dispatch-time allowlist backstop through `dispatchMacroTool`. |
| T-135-12 | Elevation of Privilege | `fq.call_macro` | mitigate | Treat as `unknown_tool` because it is absent from the in-macro registry. |
| T-135-13 | Elevation of Privilege | template masquerade | mitigate | Return `template_masquerade_tools_not_callable_from_macro` for masqueraded tool names. |
| T-135-14 | Elevation of Privilege | delegated `fq.call_model` | mitigate | Return `forbidden_tools` with `recursive_model_excluded_from_delegated_macros`. |
</threat_model>
