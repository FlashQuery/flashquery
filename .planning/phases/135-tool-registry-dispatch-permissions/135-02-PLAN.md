<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| macro AST -> ToolRegistry | User-authored server/tool names are resolved against an internal registry. |
| ToolRegistry -> native handlers | Macro dispatch crosses into privileged FlashQuery tool handlers. |
| ToolRegistry -> broker handlers | Future broker handlers cross an external MCP server boundary through `ToolFn`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-135-06 | Elevation of Privilege | `buildToolRegistry` | mitigate | Use `resolveHostToolExposure` for host calls and `assembleNativeToolRegistry` for delegated calls; omit `call_macro` from `fq` registry entries. |
| T-135-07 | Tampering | `dispatchMacroTool` | mitigate | Check `allowlist.has("server.tool")` immediately before handler invocation and return `forbidden_tools` without calling the handler. |
| T-135-08 | Elevation of Privilege | native handler access | mitigate | Forbid direct imports from `src/mcp/tools/*`; only use catalog handlers captured by `wrapServerWithToolCatalog`. |
| T-135-09 | Spoofing | broker server entries | accept | v0 `NullMcpBroker` provides no live brokered entries; future real broker authentication is out of Phase 135 scope. |
</threat_model>
