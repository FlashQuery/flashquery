<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| macro source -> evaluator | Parsed but untrusted macro AST controls evaluator flow and variable names. |
| invocation -> invocation | Each macro run must not share mutable scope or context objects with another run. |
| evaluator -> injected builtins | Test stubs now, production builtins later; builtin return values are untrusted macro values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-132-01 | I/T | `Env` and `createInvocationContext` | mitigate | Use fresh `Env` and context per `evaluateProgram`; no module-level mutable scope. |
| T-132-02 | T | `ForLoop` iterator binding | mitigate | Bind iterator with `setLocal` only; tests T-U-071 and T-U-072 reject outer variable overwrite. |
| T-132-03 | D | `WhileLoop` execution | mitigate | Call `context.checkCancelled` before statements and loop iterations; later plans add explicit cancellation tests. |
| T-132-04 | E | POC singleton `taskRegistry` pattern | mitigate | Do not import or recreate singleton task registry; task fields live on invocation context only. |
</threat_model>
