## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP session -> macro registry | A caller can only see/cancel tasks for its own explicit session identity. |
| Macro source -> native handler dispatch | Cancellation must stop before post-cancel tool dispatch side effects. |
| In-process host code -> task registry | Host-internal cancel requests must still be session-checked for defense in depth. |

| T-136-01-02 | Elevation of Privilege | `MacroTaskRegistry.cancel` | mitigate | T-U-186 asserts cross-session cancellation is refused and does not mutate the target task. |
| T-136-01-03 | Information Disclosure | terminal task records | mitigate | T-U-173/T-U-174/T-U-175 assert completed/failed/cancelled records are removed immediately. |

</threat_model>
