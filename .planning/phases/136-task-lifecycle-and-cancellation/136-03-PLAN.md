<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Macro code -> side-effecting tool dispatch | Cancellation must stop before dispatch when observed at the pre-dispatch safe point. |
| In-flight native handler -> macro evaluator | A running tool call is trusted to finish; cancellation resumes at the next evaluator safe point. |
| Cancellation signal -> response envelope | Expected cancellation must not be reported as `isError:true` runtime failure. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-136-03-01 | Tampering | `evalToolCall` | mitigate | Move check after arg evaluation and before dispatch; T-U-179 proves handler is not invoked after cancellation. |
| T-136-03-02 | Repudiation | cancellation result envelope | mitigate | Return `error:"cancelled"` with `task_id` and `at_safe_point`; T-U-183 proves canonical evidence. |
| T-136-03-03 | Denial of Service | `sleep`/`slow_op` | mitigate | Keep 100 ms chunk checks; T-U-182 proves long-running builtins observe cancellation. |
| T-136-03-04 | Tampering | post-cancel execution | mitigate | T-U-178/T-U-180/T-U-181/T-U-184 prove no post-safe-point statement, loop, pipeline, or tool side effects. |
</threat_model>
