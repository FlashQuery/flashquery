| archive_document/remove_document concurrency | Archive and remove can otherwise race over the same document lifecycle. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-130-06 | T/D | `archive_document` | mitigate | Acquire the standard `documents` write lock before mutation when locking is enabled. |
| T-130-07 | T | `archive_document` error paths | mitigate | Release the lock in `finally` so runtime failures do not leave stale active locks until TTL. |
| T-130-08 | D | Lock contention response | mitigate | Return canonical `conflict` with `details.reason: "lock_contention"` immediately when acquisition fails; do not mutate. |
| T-130-09 | T | `archive_document`/`remove_document` race | mitigate | Integration coverage uses a deterministic held-lock proxy to prove both handlers serialize through the shared `documents` lock and require the SUMMARY to document why direct concurrent timing was not used. |
| T-130-10 | I | Scope overreach | mitigate | Do not change archive payloads, source resolution, macro execution, or lock schema; only add the missing lock lifecycle. |
