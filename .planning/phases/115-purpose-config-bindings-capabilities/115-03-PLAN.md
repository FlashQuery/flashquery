<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML config -> Supabase | User-authored bindings become durable authorization rows. |
| Runtime/API binding -> Supabase | Internal runtime calls create API-owned rows that can expose template tools. |
| Binding identifier -> vault path | Ergonomic identifiers cross into filesystem-relative template identity. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-115-09 | Elevation of privilege | Runtime binding service | mitigate | Call shared capability admission before inserting API-owned rows. |
| T-115-10 | Tampering | Generic sync | mitigate | API-owned unique slots are never overwritten by YAML; warning names exact binding identity. |
| T-115-11 | Tampering | Template path normalization | mitigate | Normalize to vault-relative `template_path` and reject vault escapes through existing document/path checks. |
| T-115-12 | Denial of service | Dangling bindings | accept | Dangling but structurally valid paths warn and persist; discovery/dispatch filtering is covered in later phases. |
</threat_model>
