<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP caller -> record tools | Concurrent callers can trigger reconciliation through read and write record operations. |
| Record tools -> plugin reconciliation service | Reconciliation may classify and execute non-idempotent actions. |
| Coordination helper -> PostgreSQL session | Advisory lock correctness depends on session-stable checkout and release. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-157-04 | Tampering | records reconciliation preamble | mitigate | Serialize `reconcilePluginDocuments` and `executeReconciliationActions` with `withPluginCoordinationLock` keyed by instance/plugin/instanceName. |
| T-157-05 | Tampering | accidental global records lock replacement | mitigate | Helper key must include `config.instance.id`, `pluginId`, and `pluginInstance`; grep rejects resource literal `"records"` in record lock paths. |
| T-157-06 | Denial of Service | advisory lock release | mitigate | Release `pg_advisory_unlock` in `finally` on the same `withPgClient` session. |
| T-157-07 | Repudiation | missing concurrency decision | mitigate | Create `157-RECONCILIATION-AUDIT.md` and handler comments that reference REQ-023 and the audit artifact. |
| T-157-SC | Tampering | package installs | accept | No package-manager installs are planned for this plan. |
</threat_model>
