<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP registration to delegated provider schema | Internal MCP schemas become model-visible tool definitions. |
| Zod schema to JSON Schema | Validation schemas cross from TypeScript runtime into provider API JSON. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-116-05 | Tampering | `wrapServerWithToolCatalog` | mitigate | Wrapper delegates to the original `registerTool` and only records metadata; unit tests and server wiring grep ensure no handler behavior changes. |
| T-116-06 | Information Disclosure | `toOpenAiToolDefinition` | mitigate | Translate only input schema, name, and description; never include handler functions, config values, or secrets in provider tool definitions. |
| T-116-07 | Elevation of Privilege | strict schema normalization | mitigate | Strict mode sets `additionalProperties: false` and required keys so delegated models cannot smuggle undeclared arguments through broad schemas. |
| T-116-08 | Denial of Service | malformed schema conversion | mitigate | Conversion helper handles raw shape and Zod object inputs deterministically and throws tested errors for unsupported root schemas. |
</threat_model>
