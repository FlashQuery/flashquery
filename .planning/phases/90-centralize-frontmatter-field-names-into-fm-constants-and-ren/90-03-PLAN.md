The repairFrontmatter function builds a frontmatter object to write back. It currently uses:
```typescript

title:         frontmatter.title ?? existingDoc.title ?? filename,
created:       (frontmatter.created as string) || existingDoc.createdAt,
status:        (frontmatter.status as string) || 'active',
fqc_instance:  instanceId,
```
Change ALL of these to FM constants:
```typescript

[FM.TITLE]:    frontmatter[FM.TITLE] ?? existingDoc.title ?? filename,
[FM.CREATED]:  (frontmatter[FM.CREATED] as string) || existingDoc.createdAt,
[FM.STATUS]:   (frontmatter[FM.STATUS] as string) || 'active',
[FM.INSTANCE]: instanceId,
```

BOUNDARY — do NOT change these (they are database column references, not frontmatter keys):
- SQL string `AND column_name = 'fqc_id'` (in ensureLastSeenColumn or similar)
- Any `.eq('fqc_id', ...)` Supabase query call
- Any `INSERT INTO ... fqc_id ...` SQL string
- Variable names like `fqcId` (camelCase) — these are NOT frontmatter key strings
