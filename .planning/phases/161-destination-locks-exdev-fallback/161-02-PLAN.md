## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem rename to fallback | OS rename errors decide whether the fallback path runs. |
| move_document to durable write primitive | Source bytes cross into the destination commit primitive. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-161-03 | Tampering | EXDEV fallback destination | mitigate | Delegate destination commit to `writeVaultFile`, which uses temp-write, fsync, rename, and directory fsync. |
| T-161-04 | Repudiation | fallback failure path | mitigate | Preserve runtime error propagation when durable commit fails; do not unlink source after failure. |
| T-161-SC | Tampering | package installs | accept | No package installs are planned for this phase. |
</threat_model>
