<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Markdown/frontmatter -> graph writes | User-authored vault files decide processing level and link targets. |
| Chunk diff -> graph tables | Persisted chunk changes drive graph node/edge mutation. |
| Markdown links -> graph edge metadata | Untrusted link text becomes structured diagnostics and references. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-171-06 | Information Disclosure | structural graph writes | mitigate | Every graph write helper must take `instanceId` and filter/upsert by `instance_id`. |
| T-171-07 | Denial of Service | link resolver | mitigate | Use AST/fence-aware parsing and bounded per-document link processing; do not recursively traverse links during writes. |
| T-171-08 | Denial of Service | disabled or `fq_processing: embedded|none` | mitigate | Short-circuit before graph mutation and before pending edge queueing; tests assert no hidden graph work/cost. |
| T-171-09 | Tampering | unresolved links | mitigate | Store unresolved diagnostics only; never create fake graph nodes for nonexistent target documents or anchors. |
| T-171-10 | Information Disclosure | graph write errors | mitigate | Log bounded messages only; MCP-facing callers must receive existing expected/runtime envelopes without raw DB URLs or stack payloads. |
| T-171-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages are installed in this plan; use existing markdown parser dependencies. |
</threat_model>
