<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML config/sidecars -> runtime graph config | Operator-controlled files become validation contracts for graph writes and reads. |
| Runtime -> Postgres DDL/schema verification | Startup creates tables and constraints that protect persisted graph data. |
| Template variables -> prompt text | Namespaced graph variables render into LLM prompt input; unknown variables must not break or leak internals. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-171-01 | Information Disclosure | `fqc_graph_*` tables | mitigate | Require `instance_id` columns and indexes in all graph tables; downstream queries must filter by `config.instance.id`. |
| T-171-02 | Denial of Service | graph traversal schema/RPC groundwork | mitigate | DDL/read helper contracts must support bounded `max_depth`, `limit`, and visited-set traversal in later plans. |
| T-171-03 | Tampering | relation and prompt sidecars | mitigate | Validate relation names, directionality, detection method, confidence, metadata schema, and prompt variables before graph workers/tools run. |
| T-171-04 | Denial of Service | disabled graph mode | mitigate | `graph.enabled:false` must use explicit disabled/null graph config with no graph writes, queueing, or hidden LLM work. |
| T-171-05 | Information Disclosure | MCP error envelopes | mitigate | Foundation code must use existing bounded JSON response helpers; runtime failures must not expose database URLs, API keys, raw LLM output, or prompt bodies. |
| T-171-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages are installed in Phase 171; existing dependencies only. |
</threat_model>
