<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client -> plugin admin tools | Untrusted callers can register/unregister plugin schemas and request metadata. |
| Plugin schema -> database DDL | Plugin YAML creates or migrates dynamic tables. |
| Plugin info -> MCP client | Schema/status payloads may disclose more than requested. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-126-06 | Tampering | `unregister_plugin` | mitigate | Block unregister with live records unless `force:true`; count live rows before cleanup and return `conflict` with `isError:false`. |
| T-126-07 | Information disclosure | `get_plugin_info` | mitigate | Default to table names only; gate full `schema` and `status_detail` behind include enum. |
| T-126-08 | Denial of service | `register_plugin` schema migration | accept | Existing safe/unsafe schema migration checks remain; this plan only changes output envelope. |
| T-126-09 | Repudiation | Plugin admin responses | mitigate | Return `registered_at`, `unregistered_at`, `was_new`, and warnings structurally for auditability. |
</threat_model>
