<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client -> pending-review tool | Untrusted callers list or clear pending workflow rows. |
| Pending-review row IDs -> deletion query | User-provided IDs select rows to delete. |
| Scenario harness -> MCP server | Public contract validation executes against managed servers and test data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-126-20 | Tampering | `clear_pending_reviews` | mitigate | Clear by pending-review primary key `id`, not document `fqc_id`; scope all deletes by `instance_id` and optional `plugin_id`. |
| T-126-21 | Information disclosure | `clear_pending_reviews(action:"list")` | mitigate | Return only `{ id, type, plugin_id, table, path }`, not full context JSON or record data. |
| T-126-22 | Repudiation | clear action | mitigate | Return `{ cleared, items }` with cleared row identifiers. |
| T-126-23 | Spoofing | nonexistent IDs | mitigate | Return `warnings:["no_matching_items"]` and `cleared:0`; do not report success on nonexistent requested IDs without warning. |
| T-126-24 | Information disclosure | scenario outputs | accept | Tests run against controlled fixtures; no production secrets are asserted or printed. |
</threat_model>
