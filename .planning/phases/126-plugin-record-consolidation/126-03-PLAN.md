<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client -> `write_record` | Untrusted dynamic record write requests. |
| `write_record` -> plugin table | Validated user data is persisted to dynamic Supabase tables. |
| `write_record` -> reconciliation | Write side effects may expose pending workflow metadata. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-126-10 | Tampering | `write_record(mode:"create")` | mitigate | Validate required fields and reject generated/unknown fields before insert. |
| T-126-11 | Tampering | `write_record(mode:"update")` | mitigate | Require compound `(plugin_id, table, id)` and validate partial data before update. |
| T-126-12 | Information disclosure | `write_record` output | mitigate | Default to identification-only; include `data` and `schema_metadata` only when requested. |
| T-126-13 | Repudiation | Write side effects | mitigate | Return `reconciliation` and `pending_review` structurally when non-empty, not as prose. |
| T-126-14 | Elevation of privilege | Dynamic table scope | mitigate | Resolve table through `pluginManager.getTableSpec` and `resolveTableName`; never accept raw physical table names. |
</threat_model>
