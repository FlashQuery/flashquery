<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client -> record read/search/archive | Untrusted callers request dynamic table reads, search, and lifecycle changes. |
| Record search -> plugin tables | Queries traverse dynamic plugin tables and may expose data. |
| Archive request -> dynamic table mutation | Batch lifecycle mutation updates plugin-owned rows. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-126-15 | Information disclosure | `get_record` | mitigate | Respect `include: []` and `include: ["schema_metadata"]`; default data only for get per product contract. |
| T-126-16 | Tampering | `archive_record` | mitigate | Scope each target by resolved plugin table and `instance_id`; preserve per-element errors. |
| T-126-17 | Information disclosure | `search_records` | mitigate | Gate `data` behind include and omit unrequested schema/table details beyond required identification. |
| T-126-18 | Denial of service | all-taggable search | mitigate | Use existing max result limits per table and return bounded result envelopes. |
| T-126-19 | Repudiation | batch archive | mitigate | Preserve input order and return per-target success/error details. |
</threat_model>
