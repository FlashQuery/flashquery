<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client to discovery/help builders | Untrusted resolver params can request metadata without LLM execution. |
| Vault/template metadata to discovery output | Template frontmatter diagnostics are exposed, but template/document bodies must not be indexed or leaked by search. |
| Config capability declarations to model diagnostics | Misleading unknown-vs-false states can cause unsafe or confusing Mode 2 admission decisions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-119-01 | Elevation of Privilege | `src/mcp/tools/llm.ts` resolver dispatch | mitigate | Keep `help`, `list_models`, `list_purposes`, and `search` before model/purpose execution, reference hydration, trace snapshots, and usage writes. |
| T-119-02 | Information Disclosure | `src/llm/discovery-content.ts` search | mitigate | Index metadata keys/names/descriptions/diagnostics only; acceptance grep blocks template/document body indexing. |
| T-119-03 | Tampering | Capability diagnostics | mitigate | Reuse `modelCapabilitiesWithDefaults` semantics and test `unknown_declaration` separately from `declared_unsupported`. |
| T-119-04 | Denial of Service | `list_purposes` registry assembly | accept | Discovery may assemble existing native/template registries per purpose, matching current Phase 118 behavior; no new unbounded scan beyond existing vault template discovery path. |
</threat_model>
