| document lock key -> Postgres | File-derived lock key is transformed into SQL parameters for advisory-lock queries. |
| FlashQuery process -> Postgres session | A checked-out PoolClient holds cross-process exclusion until explicit unlock or session end. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-158-02 | Denial of Service | Tier 2 PoolClient ownership | mitigate | Release advisory locks in `finally` on the owning PoolClient before releasing Tier 1; T-U-004/T-I-004 prove release and session-end cleanup. |
| T-158-03 | Elevation of Privilege | transaction-scoped advisory locks | mitigate | Forbid `pg_advisory_xact_lock`; use session-scoped `pg_advisory_lock` only through `withPgClient`. |
| T-158-SC | Tampering | npm/pip/cargo installs | accept | No package installs are planned; existing `pg`, `async-mutex`, and `vitest` dependencies remain unchanged. |
</threat_model>
