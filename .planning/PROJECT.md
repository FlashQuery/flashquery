# FlashQuery Core

## What This Is

FlashQuery Core is an open source, local-first data management layer for individual users that sits between their AI tools (any LLM, via MCP) and a unified data store. It manages memory, documents, relational data, and vector embeddings ‚Äî all the data generated by AI-native workflows ‚Äî in a system the user owns and controls. It runs locally as a server process, exposes a single MCP server to any AI tool via stdio, stores relational data and vectors in the user's own Supabase instance, and manages a local markdown folder structure that doubles as an Obsidian vault.

**v2.9 shipped (2026-04-25).** Filesystem Primitive Tools milestone complete: `create_directory` MCP tool added, `list_files` upgraded to `list_vault` with enriched metadata and flexible output formats, `remove_directory` migrated to `files.ts`, plugin documentation updated across fq-base and fq-skill-creator. 7 phases (91-97), 13 plans, 38 requirements ‚Äî all satisfied. 1,199 unit / 47 integration tests passing.

## Context

**Problem:** Every AI tool maintains isolated memory. Claude has its memory, ChatGPT has its memory ‚Äî switch tools and start from zero. Beyond memory, AI workflows generate notes, decisions, documents, code, and records that live in chat threads with no durable storage.

**Market validation:** Open Brain (600+ GitHub stars in weeks, Supabase-backed memory table via MCP) proves demand. Mem0 ($24M Series A) proves the investment thesis. Neither solves the full problem: Open Brain is one table, Mem0 is memory-only and cloud-SaaS. Nobody provides a unified, local-first data management layer for all AI-generated data types.

**Strategic goals:** Solve the real problem for individual users; gain exposure for FlashQuery brand; attract investment (broader thesis than Mem0's $24M memory-only raise); demonstrate the "dissolving app" thesis; create natural upgrade path to FlashQuery Enterprise.
